What it does
Daily routing, automated

Real road network

Calculations on OSRM with real road graphs — not straight-line distances. Travel time accounts for the actual road network of the client's region.

Time windows and capacity

Customer delivery windows, vehicle capacity, driver shift hours. Constraints honored, not approximated.

Fleet rebalancing

When a zone overloads, orders are shifted between drivers. Workload distributes evenly across the fleet instead of piling on one route.

Warm-start re-routing

Order changes mid-day don't trigger a full recalculation. The engine adjusts the existing plan — drivers get a correction, not a new route.

Dispatcher panel

Web UI for operators: map view, plan vs fact, fleet load, route adjustments, exception handling.

Driver communication

Route delivery to drivers via Telegram or a mobile app. Status updates flow back to the dispatcher panel in real time.

How it works
Three modes — production-tested

Benchmarks on a 400-point, 13-vehicle daily delivery scenario:

Mode 1

Build from scratch

New order set every day. Full distribution across the fleet, with windows and capacity.

400 points → 5–30 sec. All orders served.
Mode 2

Re-route on changes

Plan exists, some orders changed. Minimal corrections, drivers stay on familiar routes.

Warm start: +7.7% better mileage vs from scratch.
Mode 3

Re-sort fixed assignments

Driver-customer assignments preserved. Visit order optimized within each route.

−7% mileage from re-sorting only.
One engine: OR-Tools (Google) + OSRM. 400×400 distance matrix in ~2 seconds. Handles country-scale road graphs.
